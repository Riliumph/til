# C/C++におけるswitch文のバイパスエラー

case文は当然ながらスコープを持たない。  
そのため、case文内で変数宣言をすると面白いことができる。  

``` C++
switch(x){
  case 1:
    int a = 0;
    break;
  default:
    a = 10;
    break;
}
```

こういう風に、case1の時に用意される変数だが、defaultへ飛んだ時に値が用意されていない変数に値を設定することができてしまう。  
Visual Studioではこれはバイパスエラーとされ、正直エラーだけ見ても分からない。  
少なくとも、分からなかった。
