# VSCodeとVPN

Surfsharkを使っているとWSL2条からネットワークエラーが返ることがある。  
VSCodeからjupyter-notebookなどを使っていると大変不便である。

## VPNが問題か？

VPN未使用時は問題なく通信できている模様。  

VPNを無効化すると

```console
$ ping google.com
PING google.com (142.250.76.46) 56(84) bytes of data.
64 bytes from maa03s36-in-f14.1e100.net (142.250.76.46): icmp_seq=1 ttl=117 time=3.20 ms
64 bytes from lckixa-ar-in-f14.1e100.net (142.250.76.46): icmp_seq=2 ttl=117 time=2.91 ms
^C
--- google.com ping statistics ---
2 packets transmitted, 2 received, 0% packet loss, time 1002ms
rtt min/avg/max/mdev = 2.907/3.051/3.195/0.144 ms
```

VPNを有効化すると

```console
$ ping google.com
ping: google.com: Temporary failure in name resolution
```

間違いなくVPNが悪さをしている。  
エラー自体は名前解決に失敗していることが分かる。

## 公式ドキュメントを読む

[VPN に接続されると、WSL からネットワークに接続できない](https://learn.microsoft.com/ja-jp/windows/wsl/troubleshooting#wsl-has-no-network-connectivity-once-connected-to-a-vpn>)

> Windows で VPN に接続した後、bash のネットワーク接続が切断される場合は、bash 内からこの回避策を試してください。  
> この回避策により、/etc/resolv.conf を使用して DNS 解決を手動で上書きできます。

なるほど。クリティカルに解決方法が見つかった。  
DNSサーバーを入れろとな。

## 現在の設定を確認する

```console
$ sudo su
# vi /etc/resolve.conf
```

```resolve.conf
# This file was automatically generated by WSL. To stop automatic generation of this file, add the following entry to /etc/wsl.conf:
# [network]
# generateResolvConf = false
nameserver 10.255.255.254
```

コメントから、この`resolve.conf`はWSL2の力によって自動的に生成されたことが分かる。  
`10.255.255.254`はWSL2内部専用の仮想アドレスである。  
実際はWindows自体にリダイレクトしているだけである。

## 設定を追記する

Windows側で設定されていたVPNのDNSをresolve.confに設定する。

```resolve.conf
# This file was automatically generated by WSL. To stop automatic generation of this file, add the following entry to /etc/wsl.conf:
# [network]
# generateResolvConf = false
nameserver 10.255.255.254
nameserver 151.236.14.64
nameserver 194.156.228.111
```

## この設定は自動化できない

WSL2では起動時にWindowsのresolve.confを参照して設定を自動的に生成する。  
ただし、WSL2側から今まさにVPNが有効かどうかはわからない。
